# 选型与评测框架

## 一页评测表

每个工具用一页表格评测，避免“试用即遗忘”。建议维度：

- 目标任务：它帮你完成什么，输出物是什么
- 输入成本：提示词、素材准备、上下文整理的成本
- 输出质量：一次成功率、可控性、可复用性
- 速度：从想法到可发布产物的时间
- 集成：能否接入你现有的内容、项目、自动化系统
- 合规与风险：版权、隐私、团队协作可控性
- 成本结构：订阅、按量、团队席位，是否可预测

## 最小工具栈（建议起步）

- 一个“通用入口”：用于写作、总结、脑暴与问题拆解
- 一个“设计/图像入口”：用于封面、视觉探索与素材
- 一个“剪辑/音频入口”：用于短视频或播客工作流
- 一个“开发入口”：IDE 辅助与代码生成
- 一个“自动化入口”：把重复动作固化成流程

## 30 分钟试用流程（每个工具一样）

1. 选 1 个你真实要交付的任务（不要选“演示任务”）
2. 设定通过标准：可发布、可复用、可解释
3. 记录 3 次迭代：第一次提示词，第二次修正，第三次固化模板
4. 输出“可复用资产”：提示词模板、素材清单、发布检查表
